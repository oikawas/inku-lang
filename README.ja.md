# inku

**視覚をデザインするために、万人が使える、小さく限定的な言語**

`inku` は、短く、誰でも書ける短い記述から、抽象的なベクターグラフィックを生み出す、記述ベースの描画言語です。三つの伝統に根ざしています。

- **Sol LeWittの指示書きによるアート** — 記述そのものが作品であるという思想
- **盆栽** — 空間や素材への厳格な制約が、表現を制限するのではなく凝縮する実践
- **短歌の形式** — 型が自我を削ぎ、主張ではなく提示が残る

語彙・プリミティブ・座標空間への制約は、限界ではなく、意図を可視化するための道具立てです。

---

## コンセプト

```
記述（DDLテキスト）  →  楽譜（JSON Score）  →  演奏（SVG）
人間が書く              LLM が解釈する         機械が描く
```

記述は永続し、演奏は一回性。出力は都度わずかに異なる——モデルの進化や差異が揺らぎを生み出すことを意図した設計です。
コンピューティングの技術を使いつつ、同じ記述から毎回少し（あるいは大きく）違うものが生まれます。

---

## 例

**記述:**

```
細かく揺れるペンシルの破線が3本、画面を横切る
```

**正規化DDL（第一段階の解釈後）:**

```
ペンシルで 破線を 横に 3本 置く
揺れ: 細かい
```

**JSON Score（第二段階の構造化後）:**

```json
{
  "instructions": [
    {
      "primitive": "line",
      "style": "dashed",
      "from": [0.0, 0.33],
      "to": [1.0, 0.33],
      "weight": "pencil",
      "variation": {
        "amplitude": "fine",
        "frequency": "high",
        "quality": "perlin",
        "dimensions": ["position_y"]
      }
    }
  ]
}
```

レンダラーがこの楽譜を演奏（描画）します——毎回わずかに違う形で。
誰かの指示（歌）を書き換えても良いでしょうし、ベクターグラフィックですから、大きく引き延ばして額に飾っても、壁にペイントしても良いでしょう。物理世界でのサイズの制約は有りません。

## アーキテクチャ

inku は**二段階変換パイプライン**を採用しています。

1. **解釈段階**（高度な最新モデルを想定、APIベースのモデルアクセスやローカルGPU環境）— 記述者の母語で書かれた自由な記述を読み取り、コア語彙のみで構成された正規化DDLに変換する
2. **構造化段階**（軽い枯れたモデルを想定、ローカルLLMを想定）— 正規化DDLを妥当なJSON Scoreに変換する

この分離により、各段階がそれぞれ得意なことに集中できます。解釈段階は自然言語の創造的・連想的な読み取りを担い、構造化段階は規則遵守的な出力生成を担います。

## コア語彙（歳時記 / Saijiki）

参照語彙辞書は **歳時記（Saijiki）** と呼びます。俳句の実践から借りた言葉で、季語を集めた書のこと。常に開くものではなく、必要なときに参照するものです。

| カテゴリ（日） | カテゴリ（英） | 語彙 |
|---|---|---|
| かたち | forms | 円、楕円、三角、四角、線、弧 |
| てざわり | touches | ペン、筆、クレヨン、チョーク、縄 |
| うごき | motions | 置く、並べる、埋める、散らす |
| ばしょ | places | 上、下、中心、端、隅 |
| つらなり | continuity | 実線、破線、点線、一点鎖線 |
| ゆらぎ | movements | 細かく、大きく、ゆっくり、速く、揺れる、波打つ、震える、滲む |
| いろ | colors | 白、黒、青、赤、緑、灰 |
| わりあい | proportions | 縦長、横長、全幅、半幅、半円、上弦、下弦、三日月 |

物理的・観察的な言葉のみが許容されます。「美しく」「繊細に」「激しく」といった感情的な評価はコアに含まれません。

## 設計原則

1. 記述は人間が読める——自然言語と記述言語の中間に位置する
2. 揺らぎは仕様であり、バグではない
3. 感情語彙を排除し、物理と運動の語彙を採用する
4. 固定のキャンバスを持たない——座標は 0.0〜1.0 の比率で、あらゆる媒体にスケール可能
5. 出力は静止画——動くのは見る人であり、画像ではない
6. 入力は制約付きDSLであり、完全な自由形式ではない

設計の詳細な根拠は [SPEC.md](./SPEC.md) を参照してください。

## 機能概要 (v1.2)

Web版は稼働中です。現在対応している主な機能：

- **二段階パイプライン** — 解釈（高度LLM）＋ 構造化（軽量LLM）、演奏中のステージをリアルタイム表示
- **プリミティブ** — 線・円・楕円・弧・四角・三角、各々に素材・色・スタイル・揺らぎ指定可
- **配置** — 横並べ・縦並べ・放射状・散布のレイアウト、最大1,000個、配色サイクル
- **揺らぎ** — perlin / wave / pink / white ノイズを位置軸に適用
- **背景色** — キャンバス背景を任意の色に指定可
- **閉形状の自動塗りつぶし** — 色指定のある円・楕円・四角・三角は塗りつぶし
- **わりあい語彙** — 縦長/横長の四角、全幅/半幅の線、半円・上弦・下弦・三日月の弧形状
- **バッチモード** — 改行区切りで複数記述を入力、番号付きで順次処理・進捗表示
- **履歴** — サーバー側保存、上限なし、ページネーション、モデル情報付きサムネイル
- **歳時記パネル** — 語彙一覧、クリックで記述欄に挿入

## エコシステム

関連パッケージは `inku-` プレフィックスで統一します。

- `inku-core` — コアライブラリ
- `inku-saijiki` — 語彙辞書
- `inku-nature` — Nature プラグイン（風、さざ波など）
- `inku-web` — Web UI 実装
- `inku-android` — Android 実装
- `inku-cli` — コマンドラインツール

---

## 言語バージョン

**日本語版**と**英語版**の inku は作者が維持します。他言語の実装は、OSS として各言語話者の貢献を歓迎します。

内部のJSON Score層は言語非依存（英語キーで統一）です。翻訳が必要なのは、記述の表層部分のみです。
